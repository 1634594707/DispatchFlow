# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.2.x   | Yes       |
| 0.1.x   | Best effort |

## Reporting a vulnerability

If you discover a security issue, please **do not** open a public GitHub issue.

Instead, contact the repository maintainer privately with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if available)

We aim to acknowledge reports within 7 days and provide a remediation plan where applicable.

## Scope notes

DispatchFlow is intended for development, demonstration, and learning environments. The default deployment does **not** include authentication or authorization. Do not expose an unmodified instance to the public internet without adding appropriate access controls.
