# DispatchFlow 产品路线图

> 基于主流无人车调度平台行业对标分析，结合项目现状制定。
> 
> **文档创建**：2026-05-29
> **调研范围**：经纬恒润、新石器 Neolix、Cyngn Insight、Meili FMS、Geek+ 极智嘉、海康机器人、快仓 Quicktron、Forza Fleet Manager、AWS IoT RoboRunner、Open-RMF

---

## 一、行业对标摘要

| 平台 | 核心特色 | 对 DispatchFlow 的启发 |
|------|---------|----------------------|
| **经纬恒润** | 7 大模块：业务对接、车辆调度、作业管理引擎、交通调度、一体化监控、运营数据、灰度系统 | 最完整的港口 FMS 架构参考；灰度系统支持多套调度策略 |
| **新石器 Neolix** | AI 智能体 NeoClaw（语音操控车队）、批量指令、运营数据分析 | AI 赋能运营的前沿方向；语音/自然语言交互降低操作门槛 |
| **Cyngn Insight** | 批量任务编排、可配置仪表盘、自动指令触发（低电量自动回充） | 批量操作 + 自动化规则是效率提升关键 |
| **Meili FMS** | 任务管理、交通控制、路线规划、用户权限、IoT 集成、充电管理、审计日志、紧急停止、2FA | 功能最全面的通用 FMS，适合对标补全功能矩阵 |
| **Geek+ 极智嘉** | WES + RMS + IOP 三件套、订单预测、动态理货、策略配置、实时监控告警 | 软件系统分层架构（执行层 + 管理层 + 运营层）值得参考 |
| **海康机器人** | iWMS + RCS、AI 调度算法、多类型机器人混编、统计数据看板 | AI 预测调度（提前派车到任务点等待）是效率天花板 |
| **快仓 Quicktron** | 仿真平台、数据平台、智能调度、多机器人协同 | 仿真验证调度方案的思路（我们已有仿真适配器） |
| **Forza Fleet Manager** | 实时监控、VDA5050 协议、MES 集成、预测性维护、安全防撞 | 预测性维护是长期必做项 |

### 行业共性功能（所有 FMS 平台必备）

1. 智能调度与任务管理
2. 实时车辆监控
3. 路径规划与交通管理
4. 充电/能量管理
5. 告警与异常管理
6. 数据分析看板
7. 用户权限与审计
8. 基础设施管理（地图/站点/充电桩）
9. 外部系统集成（WMS/ERP/MES）
10. 批量操作能力

## 二、DispatchFlow 现状评估

### 已实现（Phase 1–3）

- [x] 订单管理（创建、查询、取消、状态流转）
- [x] 调度任务生命周期（创建 → 派单 → 执行 → 完成/失败）
- [x] 智能派车算法（距离 + 电量惩罚 + 插枪待命三维评分）
- [x] 手动派车 / 改派
- [x] 车辆状态管理（在线/离线、调度状态）
- [x] 园区路网图（可达性判断 + 路径规划）
- [x] 充电策略（SOC 门槛、满电判定、插枪待命不掉电）
- [x] 异常闭环（自动创建、分级、去重、人工处置）
- [x] 事件驱动架构（Outbox + RabbitMQ，至少一次投递）
- [x] 分布式锁防并发派车
- [x] 车辆回报 / 遥测幂等
- [x] 调度看板（4 项核心指标 + 最近异常 + 快捷操作）
- [x] 调度工作台（任务池 + 园区态势 + 异常队列）
- [x] 车辆监控大屏（Leaflet 地图 + SSE 实时推送 + 断线降级轮询）
- [x] 移动端下单页
- [x] 通知铃铛（待处理异常提醒）
- [x] 车端网关认证（独立凭证体系）
- [x] 遥测数据接入（VehicleTelemetryRequest → Redis 运行态）
- [x] 真实车端 / 仿真车端双适配器模式

### 架构优势

- 业务状态（MySQL）与运行态（Redis）分离
- 适配器模式解耦仿真与真实车端
- Outbox 模式保证事件可靠性
- 模块化单体架构，未来可按需拆分

## 三、路线图阶段

### Phase 4 — 生产级基础设施

> 行业对标：所有 FMS 平台都具备的底线能力。没有这些，系统不能上线。
> 
> 对标平台：Meili FMS（用户管理 + 权限 + 2FA）、经纬恒润（一体化监控管理）

- [x] **登录页 + 用户管理**
  - [x] 登录页（用户名密码）
  - [x] 用户列表页（创建/编辑/禁用）
  - [x] 角色分配（VIEWER / OPERATOR / ADMIN）
  - [x] 个人设置页

- [ ] **操作日志审计**
  - 独立操作日志查询页
  - 任务详情页展示操作日志时间线
  - 关键操作留痕（派单、改派、取消、异常处置）

- [ ] **园区基础设施管理**
  - 园区 CRUD
  - 站点 CRUD（取货站/送货站）
  - 充电桩 CRUD（位置、状态）
  - 停车位管理

- [ ] **车辆 CRUD + 凭证管理**
  - 新增/编辑/停用车辆
  - 车辆凭证管理（VehicleCredential）
  - 车辆详情页补全操作日志

- [ ] **批量自动派单**
  - 工作台勾选多个任务 → 一键批量派车
  - 批量取消任务
  - 对标：新石器 NeoClaw（批量指令）、Cyngn Insight（Batch upload）

---

### Phase 5 — 实时通信与告警

> 行业对标：实时性是 FMS 的生命线。
> 
> 对标平台：经纬恒润（一体化监控）、Geek+ IOP（实时监控告警）

- [ ] **全局 WebSocket/SSE 实时推送**
  - 将 SSE 推送从监控大屏扩展到全局
  - 工作台任务池实时刷新（替代 30 秒轮询）
  - 看板统计数据实时更新

- [ ] **声音告警 + 浏览器桌面通知**
  - 新异常产生时播放提示音（可配置开关和音量）
  - Web Push Notification（离开页面也能收到通知）
  - 严重异常特殊告警音
  - 对标：Geek+ IOP（邮件/企微/Teams/电话/告警灯多通道告警）

- [ ] **任务智能排序**
  - 工作台任务池按优先级 P0 > P1 > P2 > P3 自动排序
  - 等待时长越久优先级越高（老化机制）
  - 可拖拽手动调整
  - 对标：经纬恒润（车辆调度 — 综合作业计划、优先级调度）

---

### Phase 6 — 运营数据分析

> 行业对标：所有主流 FMS 都有数据看板。
> 
> 对标平台：经纬恒润（运营数据模块）、Cyngn Insight（Configurable dashboards）、Geek+ IOP（深度数据分析）

- [ ] **运营数据报表**
  - 订单完成率趋势图（日/周/月）
  - 平均任务完成时长
  - 车辆利用率（忙碌时间/总在线时间）
  - 异常类型分布图 / 异常趋势
  - 每日运营摘要

- [ ] **充电会话监控**
  - 充电中的车辆列表（开始时间、已充电时长、当前 SOC）
  - 充电桩占用状态总览
  - 充电历史记录
  - 对标：Meili FMS（Fleet Charge Management）

- [ ] **数据导出**
  - 订单/任务/异常/车辆列表导出 Excel
  - 运营报表导出 PDF
  - 对标：经纬恒润（运营数据 — 模拟视频和报告）

---

### Phase 7 — 调度策略与多园区

> 行业对标：经纬恒润的"灰度系统"支持多套调度规则；Geek+ 的 WES 支持策略配置。
> 
> 对标平台：经纬恒润（灰度系统）、Geek+ WES（Strategy Configuration）、Meili FMS（Site Settings）

- [ ] **调度策略配置**
  - 派车评分权重可视化配置（距离/电量惩罚/插枪奖励）
  - 充电策略参数调整（满电 SOC 阈值、最低可派 SOC）
  - 配置变更记录操作日志

- [ ] **多园区 UI 适配**
  - 全局园区选择器（顶栏）
  - 订单/任务/车辆列表页增加园区筛选
  - 看板数据按园区区分

- [ ] **交通管理增强**
  - 路网可视化编辑器（在线编辑道路节点和路段）
  - 路段禁用/启用（施工、维护场景）
  - 拥堵检测与预警
  - 对标：经纬恒润（交通调度 — 交通态势感知、路径规划、交通协同）、Meili FMS（Traffic Control）

---

### Phase 8 — 外部集成与扩展

> 行业对标：WMS/ERP 集成是 FMS 落地的关键。
> 
> 对标平台：经纬恒润（业务对接）、Geek+（Seamless Integration）、Forza Fleet Manager（MES Integration）

- [ ] **外部系统 API（开放接口）**
  - RESTful API 文档完善（已有 Swagger）
  - Webhook 回调（任务状态变更通知外部系统）
  - API Key 认证

- [ ] **预测性维护**
  - 车辆健康度评分（基于里程、使用时长、故障频率）
  - 关键部件寿命预测
  - 维护提醒推送
  - 对标：经纬恒润（车辆状态监控与健康管理）、Forza Fleet Manager（Predictive Maintenance）

- [ ] **车辆历史轨迹回放**
  - 选择车辆 + 时间段回放行驶轨迹
  - 异常停留点标记
  - 轨迹速度分析
  - 对标：经纬恒润（运营全过程回溯）

---

### Phase 9 — 进阶能力（长期）

> 行业对标：新石器的 AI 智能体、快仓的仿真平台、AWS IoT RoboRunner 的数字孪生。

- [ ] **AI 智能调度助手**
  - 自然语言下达指令（"把低电量的车都派去充电"）
  - 智能推荐派车方案
  - 异常自动处置建议
  - 对标：新石器 NeoClaw（AI 智能体 — 语音操控车队）

- [ ] **数字孪生**
  - 3D 园区可视化
  - 实时车辆状态叠加
  - 仿真推演（调度方案预评估）
  - 对标：AWS IoT RoboRunner（Digital Twin）、Geek+ G-Studio（3D Real-time Data）

- [ ] **键盘快捷键 + 效率工具**
  - 工作台快捷键（R 刷新、A 自动派车、M 手动派车、↑↓ 切换任务）
  - 全局搜索（订单号/任务号/车辆编号）

- [ ] **系统健康监控**
  - 后端服务状态、Redis/RabbitMQ 连接状态
  - 消息队列积压量监控
  - API 响应时间监控
  - 对标：经纬恒润（服务器资源、网络资源、软件应用、数据库、消息中间件监管）

## 四、功能对照总览

| 功能模块 | DispatchFlow 现状 | 行业标准 |
|---------|------------------|---------|
| 智能派车 | ✅ 三维评分算法 | AI 动态调度 + 强化学习 |
| 实时监控 | ✅ SSE + Leaflet | 多通道实时推送 + 3D 可视化 |
| 充电管理 | ✅ SOC 策略 | 充电规则配置 + 充电桩管理 |
| 异常闭环 | ✅ 分级+去重+处置 | 多通道告警 + AI 自动处置 |
| 用户权限 | ✅ 登录 + RBAC + 用户管理 | RBAC + 2FA + 审计日志 |
| 运营报表 | ❌ 无 | 效率看板 + 趋势分析 + 导出 |
| 基础设施管理 | ❌ 无 UI | 地图编辑器 + 站点/充电桩 CRUD |
| 外部集成 | ❌ 无 | WMS/ERP/MES API 对接 |
| 批量操作 | ❌ 无 | 批量任务编排 + 语音批量指令 |
| 交通管理 | ✅ 基础路网 | 拥堵检测 + 动态避堵 + 红绿灯联动 |
| 预测性维护 | ❌ 无 | LSTM 预测 + 维护提醒 |
| AI 助手 | ❌ 无 | NeoClaw AI 智能体 |
