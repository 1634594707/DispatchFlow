# 坐标地图-高德 ZJF：服务范围与前端样式确认

> 整理日期：2026 年 7 月 18 日  
> 参考文件：`docs/坐标地图-高德ZJF.html`、`docs/坐标基准-叠石桥家纺城.md`、`docs/无人车真实道路配送参数补齐清单.md`。  
> 本文先确认地图语义和视觉方案；未经现场门口、装卸口、道路接入节点复核，不建议直接把本文候选范围写入生产派单代码。

## 1. 对当前 HTML 的判断

当前 HTML 已经能表达“基地 / 试点范围 / 真实范围”三层概念，但存在三个问题：

1. `REAL_RANGE` 是一个很大的近似多边形，混合了家纺城、三星镇、川姜方向和物流方向，不能直接叫“真实可配送范围”。
2. `PILOT_RANGE` 是手工绘制的外包络，当前说明写着“已校验无路段遗漏”，但 HTML 本身没有道路级、建筑级和服务位级校验结果，文案应降级为“候选试点外包络”。
3. `HUB`、范围 Polygon 和未来车辆路线不是同一种数据对象：基地是 POI/运营点，范围是服务规则，路线应来自可通行道路图，不能用 Polygon 边界代替路线。

因此这份 HTML 适合作为**范围评审和产品讨论图**，暂时不应作为生产路线或自动派单的事实源。

## 2. 真实服务范围重新划分

### 2.1 范围分四层

| 层级 | 名称 | 真实含义 | 是否自动派单 |
| --- | --- | --- | --- |
| L1-CORE | 无人车核心履约区 | 当前已建模的南排、北排、代发仓、东排、快递接驳五个服务分区 | 可以，但必须绑定服务位和道路接入节点 |
| L1-CONNECTOR | 家纺城市场连接区 | 主市场、A/B 城、三期、步行街等北侧市场区域 | 暂不全量放开，只允许已核验服务位 |
| L0-COVERAGE | 产业带分析范围 | 叠石桥与川姜/志浩双中心的统计/报表覆盖 | 不允许直接下单或派车 |
| L2-EXTERNAL | 外部物流与公共资源 | 纺都大道 188 号、深国际物流园、公共充电站等外部节点 | 单独配置外部接驳任务 |

公开规划资料把叠石桥国际纺织时尚广场核心项目描述为：南至三星南大道、北至叠石桥三期核心交易区、东至震蒙大道、西至修女路/绣女路。这个范围可以作为“市场核心区”的外部事实参考，但不等于无人车可以进入每一栋楼或每一条内部通道。citeturn0search3turn0search2

### 2.2 建议在 HTML 中采用的边界语义

- **L1-CORE**：继续使用项目 V39 五个服务区 Polygon，不扩成一个大矩形。
- **L1-CONNECTOR**：以主市场、三期、步行街为参考区域，使用虚线和半透明灰青色显示，标签写“待服务位核验”，不写“可配送”。
- **L0-COVERAGE**：保留 20km 圆形/范围用于分析，但必须显示“不参与派单”。
- **L2-EXTERNAL**：用紫灰色节点和虚线连线表示，不与 L1 道路直接连成一张可执行路线图。

## 4. 坐标读取规范

前端每个地图对象都应带以下元数据：

```text
scopeCode: L1_CORE | L1_CONNECTOR | L0_COVERAGE | L2_EXTERNAL
coordSystem: GCJ-02
coordinateRole: MAP_CENTER | POI_REFERENCE | SERVICE_POSITION | ACCESS_NODE | RESOURCE_POSITION
confidence: A | B | C
mapVersion: 当前地图版本
source: AMAP_POI | FIELD_SURVEY | OPS_LEDGER | OSM | SYNTHETIC_ANCHOR
isOrderable: true | false
isDispatchable: true | false
```

禁止以下隐式判断：

- 不能仅凭 `ZJF-` 前缀决定站点是否真实。
- 不能仅凭 `area` 决定是否允许下单。
- 不能把 POI 中心点直接当成车辆终点。
- 不能把 20km 半径内的点自动当成可配送点。
- 不能把基地、充电点、服务位和订单目标点都建模成同一种 Marker。

## 5. 车辆样式设计

### 5.1 设计方向

采用“工业导航控制台”视觉方向：深色高德底图、低饱和区域层、青色执行态、黄色风险态、红色阻断态。车辆必须一眼看出**车头方向、运行状态、是否偏航、是否带任务**，而不是只显示一个蓝色小图标。

### 5.2 车辆图标结构

车辆采用俯视简化车体，不使用复杂 3D 小车：

- 车身：圆角矩形，长宽比例约 1.65:1。
- 车头：前端加入一个短三角箭头或高亮端帽，表示行驶方向。
- 车体内部：显示车辆编号短码，例如 `A07`，缩放较小时隐藏文字。
- 外圈：根据状态显示不同光环，不使用大面积发光覆盖道路。
- 车尾：可选显示载货/空载小条，不显示过多状态文字。
- 路线上：车体只显示当前车辆，历史轨迹使用低透明度尾迹。

### 5.3 车辆状态样式

| 状态 | 车体 | 光环/边框 | 运动效果 | 说明 |
| --- | --- | --- | --- | --- |
| 可派空闲 | `#20D6E8` | 青色细环 | 轻微呼吸 | 可接受新任务 |
| 执行配送 | `#2F80FF` | 蓝色方向环 | 沿路线移动 | 显示目标点和 ETA |
| 去充电 | `#A66CFF` | 紫色环 | 慢速脉冲 | 显示充电目标和剩余电量 |
| 到站装卸 | `#FFC857` | 黄色实线 | 停止闪烁 | 显示服务位占用和剩余服务时间 |
| 等待门禁/排队 | `#FF9F43` | 橙色虚线环 | 低频脉冲 | 不应被误读为正常行驶 |
| 偏航/重规划 | `#FF5C5C` | 红色双环 | 快速闪烁 | 显示偏离距离和重规划原因 |
| 离线 | `#6B7280` | 灰色无光 | 无动画 | 显示最后上报时间 |
| 人工接管 | `#F43F5E` | 红色方角框 | 固定警示角标 | 禁止自动再次派车 |

### 5.4 车辆必须关联的地图信息

点击车辆后，不只显示电量和任务，还要显示：

- 当前路线 ID。
- 当前道路路段/节点。
- 路线来源和地图版本。
- 规划路线与实际轨迹差异。
- 偏航距离。
- 当前目标服务位，而不是只显示站点名。
- 最近一次碰撞校验和重规划时间。

## 6. 下单目标配送点样式设计

### 6.1 目标点不能只用一个普通 Marker

订单目标点需要区分“任务目标”和“可停靠服务位”：

- 外层：目标区域环，表示订单要到达的服务区域。
- 中层：服务位图标，表示车辆实际停靠位置。
- 内层：任务类型符号，取货用向上箭头，送货用向下箭头，充电用闪电，待命用方块。
- 连接线：从道路接入节点到服务位，用短虚线表示最后一段接入，不画穿建筑的直线。
- 状态角标：可达、排队、占用、封闭、待核验。

### 6.2 目标点视觉编码

| 目标类型 | 主色 | 图标 | 备注 |
| --- | --- | --- | --- |
| 取货 | `#00D4A8` | 向上箭头/箱体 | 显示取货服务时长 |
| 送货 | `#3B82F6` | 向下箭头/箱体 | 显示卸货服务位 |
| 充电 | `#A66CFF` | 闪电 | 显示桩位占用和排队 |
| 待命 | `#94A3B8` | 方块/停车位 | 不作为订单目标 |
| 快递接驳 | `#F59E0B` | 交接箱 | 显示接驳时间窗 |
| 不可达 | `#EF4444` | 断开链条 | 必须显示具体原因 |
| 待核验 | `#F97316` | 问号/定位点 | 不允许自动派车 |

### 6.3 订单目标卡片

点击目标点后使用右侧轻量卡片或地图浮层，内容按决策优先级排列：

1. `任务号 + 取货/送货`。
2. 目标服务位名称和服务位编码。
3. 当前可达性：可执行 / 排队 / 被占用 / 不可达 / 待核验。
4. 预计到达时间、服务时间、预计离开时间。
5. 接入道路、进入方向和限制车辆类型。
6. 路线来源、地图版本和最后校验时间。
7. 操作：查看路线、重新规划、人工接管、标记站点问题。

禁止只显示“站点名称 + 经纬度”，那无法帮助调度员判断是否能真正停靠。

## 7. 地图图层层级

建议将当前 HTML 的三层复合为以下顺序：

1. 高德底图。
2. 建筑物/步行区/河道等不可通行遮罩。
3. L0 分析覆盖层，默认低透明度。
4. L1-CORE 可派单分区，使用低透明填充。
5. L1-CONNECTOR 待核验连接区，使用虚线边界和灰橙色填充。
6. 道路中心线/可行驶带/封路标记。
7. 规划路线和实际轨迹。
8. 车辆。
9. 服务位、订单目标点和告警浮层。

车辆和订单目标点永远压在区域 Polygon 之上；建筑遮罩不能覆盖车辆，让用户误以为车辆消失。

## 8. 最小交互验收

- [ ] 点击 L1-CORE，只显示默认可派单服务位。
- [ ] 点击 L1-CONNECTOR，明确显示“待核验，不可自动派车”。
- [ ] 点击主市场 POI，不自动生成订单目标点。
- [ ] 点击三期 POI，能看到它与 `ZJF-DROP-02` 当前服务位是否一致。
- [ ] 点击车辆，能看到车头方向、当前道路、路线来源和偏航距离。
- [ ] 规划路线失败时，车辆停止并显示失败原因，不回退为直线或示意路线。
- [ ] 目标点在建筑中心时，显示“无服务位”，不绘制可执行路线。
- [ ] 目标点被服务位占用时，显示排队/预约状态，不显示绿色可执行状态。
- [ ] 充电目标显示具体桩位，不显示公共充电 POI 中心。
- [ ] 缩放地图时，车辆图标和目标点图标保持可识别但不遮挡道路。

## 9. 最终确认结果

- **真实服务范围**：不能用当前 `REAL_RANGE` 作为自动配送范围；它应改为 L0 产业带参考范围。
- **无人车范围**：当前五个 V39 核心分区可以作为 L1-CORE 候选履约区，但仍需服务位/接入节点确认。
- **北侧市场**：主市场、A/B 城、三期、步行街应单独作为 L1-CONNECTOR，未经现场核验不自动派单。
- **L1 基点**：`[121.080354, 31.961977]` 继续作为局部投影基点。
- **L0 主市场中心**：`[121.080800, 31.964305]` 作为主市场展示中心。
- **基地**：HTML 中 `[121.080681,31.960337]` 先作为运营基地候选，需运营台账确认，不直接宣称高德 POI 已验证。
- **样式方向**：车辆采用俯视车体 + 方向箭头 + 状态光环；订单目标采用目标环 + 服务位芯点 + 接入虚线 + 可达性状态。
