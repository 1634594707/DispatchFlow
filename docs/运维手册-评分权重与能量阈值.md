# DispatchFlow 运维手册 — 评分权重与能量阈值

> 阶段八 8.3 交付物：`DispatchScoringProperties` 与 `FleetEnergyProperties` 默认值与调整指南。
> 运维可参照本手册调整派单评分权重与车辆能量阈值。

---

## 1. 派单评分权重（DispatchScoringProperties）

**配置前缀**：`fsd.dispatch.scoring`
**对应类**：`com.fsd.dispatch.config.DispatchScoringProperties`
**评分模型**：**总分越低越优先**（lower total score wins）

### 1.1 权重字段一览

| 字段名 | YAML 键 | 环境变量 | 默认值 | 单位 | 说明 |
|---|---|---|---|---|---|
| `weightDistance` | `weight-distance` | `FSD_DISPATCH_SCORING_WEIGHT_DISTANCE` | `1.0` | 系数 | 距离评分系数。车辆到取货点的距离（米）× 该系数 = 距离得分。增大该值会让派单更倾向于就近车辆。 |
| `weightSocMargin` | `weight-soc-margin` | `FSD_DISPATCH_SCORING_WEIGHT_SOC_MARGIN` | `0.15` | 系数 | 每低于满电 1% SOC 的罚分。增大该值会让派单更倾向于电量充足的车辆。 |
| `weightPluggedStandbyBonus` | `weight-plugged-standby-bonus` | `FSD_DISPATCH_SCORING_WEIGHT_PLUGGED_STANDBY_BONUS` | `80.0` | 分 | 插电待命车辆的奖励分（从总分中减去）。增大该值会让派单更倾向于已插电的待命车辆。 |
| `weightPriority` | `weight-priority` | — | `1.0` | 系数 | 订单优先级权重。高优先级订单对距离和 SOC 评分更敏感。 |
| `weightFairness` | `weight-fairness` | — | `0.5` | 系数 | 车辆均衡因子权重（闲置时间奖励）。增大该值会让派单更倾向于长时间闲置的车辆。 |
| `maxIdleBonus` | `max-idle-bonus` | — | `30.0` | 分 | 闲置时间奖励上限。防止长时间闲置车辆获得过高奖励导致派单不合理。 |
| `weightCongestion` | `weight-congestion` | — | `0.3` | 系数 | 交通拥堵因子权重。增大该值会让派单更倾向于避开拥堵路段的车辆。 |

### 1.2 调整示例

**场景 1**：试点初期希望就近派单，减少长距离行驶
```yaml
fsd:
  dispatch:
    scoring:
      weight-distance: 1.5      # 提高距离权重
      weight-soc-margin: 0.1    # 降低 SOC 权重
      weight-fairness: 0.3      # 降低均衡因子
```

**场景 2**：希望优先使用插电待命车辆（降低电池损耗）
```yaml
fsd:
  dispatch:
    scoring:
      weight-plugged-standby-bonus: 120.0  # 提高插电奖励
```

**场景 3**：希望避免低电量车辆被派单（抛锚风险高）
```yaml
fsd:
  dispatch:
    scoring:
      weight-soc-margin: 0.25   # 提高 SOC 罚分
```

### 1.3 调整建议

- 一次只调整 1-2 个参数，观察 24 小时派单效果后再决定是否继续调整
- `weightDistance` 不建议低于 0.5，否则可能派单到很远的车辆
- `weightPluggedStandbyBonus` 不建议高于 150，否则非插电车辆几乎无法被派单
- `maxIdleBonus` 不建议高于 50，否则长时间闲置车辆会获得过高优先级

## 2. 车辆能量阈值（FleetEnergyProperties）

**配置前缀**：`fsd.fleet.energy`
**对应类**：`com.fsd.dispatch.config.FleetEnergyProperties`

### 2.1 阈值字段一览

| 字段名 | YAML 键 | 环境变量 | 默认值 | 单位 | 说明 |
|---|---|---|---|---|---|
| `lowSocThreshold` | `low-soc-threshold` | `FSD_FLEET_ENERGY_LOW_SOC_THRESHOLD` | `20` | % | 地图低电量展示阈值。SOC 低于该值的车辆在地图上标记为低电量。 |
| `returnToChargeThreshold` | `return-to-charge-threshold` | `FSD_FLEET_ENERGY_RETURN_TO_CHARGE_THRESHOLD` | `20` | % | 自动回充触发阈值。SOC 低于该值的 IDLE 车辆自动进入充电队列。 |
| `criticalSocThreshold` | `critical-soc-threshold` | — | `5` | % | 危急电量驻车阈值。SOC 低于该值的车辆强制驻车并告警。 |
| `minAssignableSoc` | `min-assignable-soc` | `FSD_FLEET_ENERGY_MIN_ASSIGNABLE_SOC` | `30` | % | 最低可派单 SOC。SOC 低于该值的车辆不再派新任务。 |
| `fullSoc` | `full-soc` | `FSD_FLEET_ENERGY_FULL_SOC` | `100` | % | 满电 SOC。 |
| `chargeCompleteSoc` | `charge-complete-soc` | — | `90` | % | 充电结束并恢复派单阈值。SOC 达到该值后车辆从充电队列释放。 |
| `chargeRatePerTick` | `charge-rate-per-tick` | `FSD_FLEET_ENERGY_CHARGE_RATE_PER_TICK` | `4` | %/tick | 每个 tick 充电速率。 |
| `reserveSocFloor` | `reserve-soc-floor` | `FSD_FLEET_ENERGY_RESERVE_SOC_FLOOR` | `8` | % | SOC 保留下限（防止电池过放）。 |
| `busyDrainIntervalTicks` | `busy-drain-interval-ticks` | `FSD_FLEET_ENERGY_BUSY_DRAIN_INTERVAL_TICKS` | `8` | tick | 执行中车辆每隔多少 tick 消耗 1% SOC。 |
| `busyDrainMetersPerPercent` | `busy-drain-meters-per-percent` | `FSD_FLEET_ENERGY_BUSY_DRAIN_METERS_PER_PERCENT` | `150` | 米/% | 真实地图配送：每下降 1% SOC 约需行驶的米数（按 geo 弧长计）。 |
| `idleDrainProbability` | `idle-drain-probability` | `FSD_FLEET_ENERGY_IDLE_DRAIN_PROBABILITY` | `0.06` | 概率 | 闲置车辆每个 tick 的 SOC 自放电概率。 |
| `pluggedStandbyNoDrain` | `plugged-standby-no-drain` | `FSD_FLEET_ENERGY_PLUGGED_STANDBY_NO_DRAIN` | `true` | 布尔 | 插电待命车辆是否不自放电。 |
| `idleChargeWhenNoDemand` | `idle-charge-when-no-demand` | `FSD_FLEET_ENERGY_IDLE_CHARGE_WHEN_NO_DEMAND` | `true` | 布尔 | 无订单需求时闲置车辆是否自动充电。 |
| `energyRecoveryMode` | `energy-recovery-mode` | — | `CHARGE` | 枚举 | 能量回收模式：`CHARGE`/`SWAP`/`AUTO`。 |
| `swapDurationTicks` | `swap-duration-ticks` | — | `5` | tick | 换电时长。 |
| `chargingTimeoutMinutes` | `charging-timeout-minutes` | — | `240` | 分钟 | 充电会话最长持续时长（ALG-10 修复）。超时后强制终止。设为 0 禁用（不推荐）。 |

### 2.2 阈值关系图

```
0% ─────── 5% ─────── 20% ─────── 30% ─────── 90% ─────── 100%
  │          │           │           │           │           │
  │          │           │           │           │           └─ fullSoc（满电）
  │          │           │           │           └─ chargeCompleteSoc（充电完成，恢复派单）
  │          │           │           └─ minAssignableSoc（最低可派单 SOC）
  │          │           └─ returnToChargeThreshold = lowSocThreshold（自动回充 + 地图标记低电量）
  │          └─ criticalSocThreshold（危急电量，强制驻车）
  └─ reserveSocFloor（保留下限）
```

**关键约束**：
- `criticalSocThreshold` < `returnToChargeThreshold` < `minAssignableSoc` < `chargeCompleteSoc` ≤ `fullSoc`
- `minAssignableSoc - returnToChargeThreshold` ≥ 10（建议值），给长路径任务留 10% 安全垫

### 2.3 调整示例

**场景 1**：试点初期希望减少抛锚风险，提高安全垫
```yaml
fsd:
  fleet:
    energy:
      return-to-charge-threshold: 25    # 提前回充
      min-assignable-soc: 35            # 提高 min-assignable
```

**场景 2**：希望车辆充得更满再派单（减少中途充电）
```yaml
fsd:
  fleet:
    energy:
      charge-complete-soc: 95           # 充到 95% 再释放
```

**场景 3**：希望使用换电模式而非充电模式
```yaml
fsd:
  fleet:
    energy:
      energy-recovery-mode: SWAP
      swap-duration-ticks: 3            # 换电更快
```

### 2.4 调整建议

- `returnToChargeThreshold` 不建议低于 15，否则车辆可能在长路径任务中途抛锚
- `minAssignableSoc` 与 `returnToChargeThreshold` 至少相差 10
- `chargeCompleteSoc` 不建议高于 95，否则充电末期会过慢（锂电池特性）
- `chargingTimeoutMinutes` 不建议低于 60，否则正常充电可能被误终止；不建议设为 0（禁用超时）
- 切换 `energyRecoveryMode` 前请确认对应设施（充电桩/换电柜）已就绪

## 3. 热更新支持

| 配置组 | 是否支持热更新 | 说明 |
|---|---|---|
| `DispatchScoringProperties` | 否（需重启） | 阶段八 8.3 仅文档化，未实现 Redis 热更新 |
| `FleetEnergyProperties`（充电阈值） | 是（Redis） | 阶段八 8.4 实现，详见下文 |
| `FleetEnergyProperties`（其他字段） | 否（需重启） | 仅充电触发阈值支持 Redis 热更新 |

### 3.1 充电阈值 Redis 热更新（阶段八 8.4）

充电触发阈值支持通过 Redis 动态配置，修改后下个调度周期（默认 5s）生效，无需重启服务。

**Redis Key**：
```
fsd:config:energy:return-to-charge-threshold
fsd:config:energy:min-assignable-soc
fsd:config:energy:charge-complete-soc
fsd:config:energy:critical-soc-threshold
```

**值格式**：整数（百分比，0-100）

**修改示例**（redis-cli）：
```bash
# 临时将自动回充阈值调整为 25%
SET fsd:config:energy:return-to-charge-threshold 25

# 恢复默认（删除 Redis key 即可回退到 YAML 配置）
DEL fsd:config:energy:return-to-charge-threshold
```

**优先级**：Redis key > YAML 配置 > Java 默认值

## 4. 验证清单

调整后请验证：

- [ ] 服务启动无报错（日志中无 `BindException` 或 `IllegalArgumentException`）
- [ ] 派单评分符合预期（观察 5-10 单派单结果）
- [ ] 车辆 SOC 阈值生效（在监控大屏观察车辆状态变化）
- [ ] 充电阈值 Redis 热更新生效（修改 Redis key 后 10s 内观察车辆行为变化）
- [ ] 无抛锚告警（24h 内 `criticalSocThreshold` 告警数 ≤ 试点前水平）
