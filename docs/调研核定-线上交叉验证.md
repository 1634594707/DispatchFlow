## 0. 为什么可以"不现场走访"

路线图第 2 节的"需线下"项本质分两类：
1. **宏观实景**（站点是否真实、是否人车混行、是否有门禁窄路、是否有服务位）——这类可由公开运营报道 + 行业研究交叉验证，不必到现场。

## 2. 站点/服务位建模范式核定（替代 2.1 现场定义）

学术与工程研究一致支持"站点 ≠ 落点"的拆分：

- **无人车停靠位置选址**：以停靠位置 + 配送中心为决策变量，实证降低配送距离/时间（上海虹口生鲜无接触配送案例）。
- **投递口布局优化**：在改进 A* 中加入转弯代价与节点负载代价（负载均衡混合遗传算法），用于优化快递分拨中心投递口布局。
- **末端物流节点布局优化**：前置仓/驿站/智能柜差异化配置，P-中心模型 + NSGA-Ⅱ。

结论：**把"业务站点 / 服务位 / 接入节点 / 资源位"拆开是国际通行做法** ✅ 核定通过。路线图 2.1 的拆分建议有充分方法论支撑，不是拍脑袋。

## 3. 坐标系陷阱核定（替代 2.3A + 支撑 4.6）⚠️ 紧迫度上调

### 3.1 公开事实

- 中国公开地图强制使用 **GCJ-02（火星坐标系）**，对 WGS-84 做非线性加密偏移，**偏移量 100–700 米，且不可逆**。
- **Leaflet 默认使用 WGS-84** 渲染；高德/腾讯底图为 GCJ-02，百度再叠 BD-09。
- 直接将 WGS-84 坐标画到 GCJ-02 底图上，路线整体"漂移"数十至数百米，直接影响路线规划与轨迹回放。
- 严禁用固定 Δ 补偿（长三角误差 ±180m）；应使用 gcoord 等库（RMSE ≤ 0.8m）或官方坐标转换 API。

### 3.2 对 DispatchFlow 的具体风险（新发现）

项目前端技术栈含 **Leaflet**（`USER.md` 明确），站点表单有 `coordLng/coordLat` 标注 "GCJ-02（可选）"。若底图采用高德/腾讯 GCJ-02 瓦片，而 Leaflet 按 WGS-84 渲染、坐标未统一转换 → **路线/站点必然漂移 100–700m**。

这把路线图 4.6（坐标距离混用）从"代码内部风险"升级为"**必现的视觉+距离错误**"，且与 4.1（首尾无 GPS）叠加后，ETA 与轨迹可信度进一步崩坏。

结论：2.3A（记录 WGS84/GCJ-02 来源、不混写）+ 4.6（统一投影参数）**✅ 核定通过，且紧迫度从 P1 上调到 P0**。

## 4. MAPF 预约与资源锁定语义核定（支撑 4.3 + 4.8）

### 4.1 公开事实

- **时空预约图（TSRG）**：路径节点建模为 `{坐标 + 时间窗 + 占有者ID}`，而非静态坐标点；双向布局易出现 heading-on / loop 死锁。
- **冲突解决**：优先级通行 + 动态重规划（把冲突点当临时障碍用 A* 重规划）。
- **规模**：仓库 AGV 系统多车并发是经典 MAPF（NP-hard）场景，预约失败/冲突是首要工程风险。

### 4.2 对 DispatchFlow 的核定

- **4.3（MAPF 兜底语义）**：`reserved=false` 必须显式区分"找不到路"与"找到路但无时空资源"；失败应返回候选路线/等待，而非伪装成成功路线。**✅ 核定通过**（与路线图一致，且被死锁文献强化）。
- **4.8（服务位并发锁定）**：服务位预约必须有**原子锁 + owner_id + 时间窗 + 全链路释放**。路线图代码复核已发现 `StationServicePositionLockService` **无任何调用方、未接入派单链路**——这恰是 MAPF 文献指出的典型失效模式（双向软锁定、CONFLICT 死锁）。在找家纺 264–300+ 车、24h 高频的真实规模下，此缺口是 **P0 级真实风险**。**✅ 核定通过，风险上调**。

## 5. 空路网 / 直线兜底反模式核定（支撑 4.2 + 4.4）

行业共识：路径规划应先做**地图匹配（snapping）**把起终点吸附到道路，再进入图搜索；直线段穿越建筑/障碍物是明确反模式。空路网不应返回"可达"，而应明确不可达/未配置。

结论：**4.1 起终点吸附、4.2 空路网返回不可达、4.4 STRAIGHT_LINE 禁入执行队列 —— ✅ 核定通过**（与代码复核互证）。

### 6.2 线上核定新增的发现（代码复核时未知）

1. **Leaflet + GCJ-02 底图未统一转换 → 必现漂移**：将 4.6 紧迫度从 P1 上调 P0（见 §3.2）。

3. **真实车辆为 L4 自动驾驶（新石器）**：DispatchFlow 做的是"园区级全局调度"，车辆本地导航由车载系统负责——前端"规划路线/实际轨迹"的展示应区分"调度下发路径"与"车载实际轨迹"，避免把车载局部避障误判为路线错误。

- 每个站点的精确经纬度、具体门口/装卸口编号、服务方向、接入道路节点编码。
- 具体封路时段、门禁开启时段、限高限宽数值。
- 这些可从"高德/天地图底图 + 找家纺运营台账"导出，不必全员现场走访——但确实需要数据，不是文章能替代。
