## 二、方案确认

### 1. 场景定位

- 园区是虚拟的，不绑定真实地理坐标
- 使用园区设计图替代真实地图
- 调度核心逻辑真实，车辆位置和移动过程使用模拟数据

### 2. 前端地图方案

- 使用 Leaflet `L.CRS.Simple` 自定义坐标系
- 园区设计图作为底图，坐标范围为 `[0, 0]` 到 `[1200, 800]`
- 所有标记均使用园区自定义坐标定位
- 地图缩放建议：
  - `minZoom: -1`
  - `maxZoom: 3`

### 3. 后端坐标方案

- 复用车辆字段：
  - `currentLongitude -> X`
  - `currentLatitude -> Y`
- 不修改数据库字段名，只修改业务语义
- 模拟引擎按园区坐标计算移动、距离、路径

### 4. 园区设计图结构

园区设计图包含：

- 园区边界
- 外围环路
- 双横主路
- 四纵联络路
- 建筑/仓库/收货区
- 停车区与充电区
- 预设站点 `A1~A4, B1~B4`
- 绿化带或水域

### 8. 假数据模拟系统

#### 虚拟无人车

- 预设 `4` 辆虚拟车辆
- 初始位置分布在停车区/待命点
- 定时更新位置，在道路上随机巡逻或待命
- 电量缓慢消耗，起始值在 `80%~100%`
- 模拟偶尔短时离线

#### 虚拟订单与站点

- 预设 `8` 个园区站点
- 创建订单时从站点中选择起点和终点
- 调度算法根据车辆到起点距离，选择最近空闲车

#### 调度全链路模拟

- 派单
- 车辆从当前位置移动到取货站
- 到达取货站后停留 `3 秒` 模拟装货
- 装货完成后移动到送货站
- 到达送货站后停留 `3 秒` 模拟卸货
- 卸货完成后订单结束，车辆释放

## 三、实施顺序

1. 生成园区设计图 SVG

3. 前端地图切换为 `Simple CRS` 园区模式
4. 联调测试完整调度链路
